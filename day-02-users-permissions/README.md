Day 02 – Users & Permissions

Scenario

A new developer joins the team and needs access to a project directory without being granted full system privileges.

---

Objective

* Create and manage users and groups
* Assign proper file ownership
* Configure secure permissions

---

Actions Taken

* Created a new user account
* Created a group for shared access
* Added the user to the group
* Created a project directory
* Assigned ownership and permissions

---

Results

* User successfully accessed the project directory
* Permissions restricted unauthorized access
* Verified correct group membership and file access

---

Commands Used

See `commands.sh`

---

What I Learned

Proper user and permission management is critical for system security. Misconfigured permissions can expose sensitive data or allow unauthorized access.

---

Why It Matters

This simulates real-world system administration where users must be given access only to the resources they need—nothing more.
